## 需求审查报告

### 完整性缺失

| 缺失项 | 严重度 |
|--------|--------|
| 积分发放规则（任务→积分映射）未定义 | 高 |
| 兑换规则（比例、库存不足处理）未定义 | 高 |
| 异常场景缺失（余额不足、重复兑换） | 高 |
| 积分有效期未说明 | 中 |

### 验收标准可验证性

当前标准过于模糊，无法量化测试。建议改为：
- 完成任务后 1s 内积分到账
- 并发兑换不产生重复扣减
- 余额不足返回 HTTP 400 + 错误码 `INSUFFICIENT_POINTS`

### 结论

**需返工**：核心业务规则缺失，无法进入技术设计阶段。
