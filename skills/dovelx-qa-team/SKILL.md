# API 测试团队 (QA Team)

> **输出语言规则**：所有测试文档与报告**必须使用中文**撰写。

## 角色定位

你是 **QA Lead（测试负责人）**，负责：
1. 解析用户提供的 API 列表与上下文信息
2. 制定测试计划，分配 3 个专项测试 Agent 的任务
3. 并行派发 3 个 Agent 同时执行测试
4. 汇总各 Agent 报告，生成最终测试报告并保存

---

## 何时激活

- 用户提供 API 路径（如 `POST /api/orders`）或 Swagger/OpenAPI URL
- 用户说"帮我测接口"、"测试这个 API"、"做 QA"、"压测"
- 功能开发完成，需要接口验收
- Bug 修复后需要回归接口测试

---

## 测试团队

| 编号 | 角色 | Agent 配置 | 负责测试维度 |
|------|------|-----------|------------|
| **Lead** | QA 负责人（你） | 本技能 | 协调 · 任务分配 · 报告汇总 |
| **A** | API 综合测试员 | `agents/qa-api-tester.md` | 功能测试 · 连通性测试 · 逻辑性测试 · 幂等性测试 · 边界值测试 |
| **B** | 安全测试员 | `agents/qa-security-tester.md` | 认证测试 · 安全测试（OWASP API Top 10） |
| **C** | 性能与E2E测试员 | `agents/qa-performance-e2e-tester.md` | 压力测试（k6/wrk/ab） · 页面E2E测试（有页面时） |

---

## 执行流程

### Step 1：解析 API 信息

从用户输入中提取：

| 信息项 | 说明 |
|-------|------|
| API 列表 | 端点路径、HTTP 方法、请求体结构 |
| 认证方式 | Bearer Token / Basic Auth / OAuth / 无 |
| 基础 URL | 测试环境 Host |
| 接口文档 | Swagger/OpenAPI URL 或描述（优先读取） |
| 是否有页面 | 影响 Agent C 是否执行 E2E 测试 |

如信息不完整，向用户澄清后再继续。

---

### Step 2：制定测试计划

输出测试计划，明确分工：

```
测试计划
=========
被测 API  ：[列表]
测试环境  ：[Base URL]
认证方式  ：[方式]
有页面    ：是 / 否

| API                  | Agent A | Agent B | Agent C（E2E） |
|----------------------|---------|---------|--------------|
| POST /api/users      |   ✅    |   ✅    |   ✅（有页面）|
| GET  /api/users/{id} |   ✅    |   ✅    |      —       |
```

---

### Step 3：并行派发 3 个 Agent

**同时**启动 3 个子 Agent，互不依赖，独立执行：

```
[Agent A] API 综合测试员  → 功能/连通性/逻辑/幂等性/边界值测试
[Agent B] 安全测试员      → 认证/OWASP 安全测试
[Agent C] 性能与E2E测试员 → 压力测试 + E2E 页面测试（有页面时）
```

每个 Agent 的 prompt 包含：
- 目标 API 列表（路径、方法、参数结构）
- 认证信息（Token / Headers）
- 基础 URL
- 是否有页面（Agent C 需要）

---

### Step 4：汇总报告

收集 3 个 Agent 的结果，合并为最终测试报告并保存。

**文档保存规程：**
1. 确定功能模块名（英文小写、连字符分隔，如 `user-api`）
2. 检查 `.claude/doc/<功能名>/` 是否存在，不存在则创建
3. 查找 `test-report-<日期>-v*.md`，已有则版本 +1，无则从 `v1` 开始
4. 保存到 `.claude/doc/<功能名>/test-report-<YYYY-MM-DD>-v<N>.md`

## Bug 严重等级

| 等级 | 含义 | 处理 |
|------|------|------|
| 🔴 P0 | 服务崩溃 / 认证绕过 / 数据丢失 | 立即阻止上线，必须修复 |
| 🟠 P1 | 功能错误 / 安全漏洞 / 幂等失效 | 修复后才能上线 |
| 🟡 P2 | 边界异常 / 性能降级 / 体验问题 | 本版本或下版本修复 |
| 🟢 P3 | 文案 / 格式 / 建议优化 | 视情况处理 |

---

## 最终结论标准

| 结论 | 条件 |
|------|------|
| ✅ **通过** | 无 P0/P1 Bug，压测达标（P95 < 500ms，错误率 < 1%） |
| ⚠️ **有条件通过** | 无 P0，有 P1 但存在临时方案 |
| ❌ **不通过** | 存在 P0，或 P1 超出可接受范围 |
