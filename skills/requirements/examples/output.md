# 任务管理功能 PRD

**版本：** v1.0
**日期：** 2026-04-13
**状态：** 待评审

---

## 一、背景与目标

### 业务背景
用户需要一套轻量的任务管理工具，以个人或小团队为主要场景，解决任务混乱、进度不透明的问题。

### 业务目标
- 提升个人/团队任务管理效率
- 降低任务遗漏率
- 提供直观的进度可视化

### 成功指标
- 任务创建到完成的平均周期缩短 20%
- 每周活跃用户任务完成率 > 80%

---

## 二、用户故事

### US-01：创建任务
**作为**一名用户，**我希望**能创建任务并填写基本信息（标题、描述、截止日期、优先级），**以便**我能追踪待办事项。

**验收标准：**
- 任务标题为必填项，最多 100 字符
- 截止日期可选，不能早于当前日期
- 优先级默认为「中」，可选高 / 中 / 低
- 创建成功后，任务出现在看板「待办」列

### US-02：更新任务状态
**作为**一名用户，**我希望**通过拖拽将任务在看板列之间移动，**以便**直观反映任务进度。

**验收标准：**
- 看板包含三列：待办 / 进行中 / 已完成
- 支持在列之间拖拽移动
- 拖入「已完成」列自动记录完成时间
- 操作后实时保存，刷新不丢失

### US-03：标记任务完成
**作为**一名用户，**我希望**能快速勾选任务为已完成，不必打开详情页，**以便**节省操作步骤。

**验收标准：**
- 任务卡片上提供勾选框
- 勾选后样式变更（灰色/删除线），并移入「已完成」列
- 支持反选（撤销完成）

### US-04：查看优先级
**作为**一名用户，**我希望**在看板上能直观识别任务优先级，**以便**优先处理重要任务。

**验收标准：**
- 高优先级：红色标记
- 中优先级：黄色标记
- 低优先级：灰色标记
- 支持按优先级筛选

---

## 三、功能范围

### 本期包含（In Scope）
- 任务 CRUD（创建、查看、编辑、删除）
- 优先级设置（高 / 中 / 低）
- 截止日期设置
- 看板视图（三列）
- 看板拖拽交互

### 本期不包含（Out of Scope）
- 多人协作 / 任务分配
- 子任务 / 任务依赖
- 邮件 / 消息通知
- 移动端 App（仅 Web）

---

## 四、非功能需求

| 指标 | 要求 |
|------|------|
| 性能 | 看板加载时间 < 1s（任务数 ≤ 200） |
| 可用性 | 服务可用性 ≥ 99.5% |
| 数据安全 | 用户数据隔离，不可跨账号访问 |
| 浏览器兼容 | Chrome 100+、Edge 100+、Firefox 110+ |
