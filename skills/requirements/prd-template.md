# PRD 模板

生成需求文档时使用此模板。路径：`.claude/doc/<功能名>/design-<日期>-v1.md`

```markdown
# <功能名称>

**需求日期**：YYYY-MM-DD
**需求来源**：[用户/业务方/产品]
**优先级**：P0 / P1 / P2

## 1. 背景与目标

### 1.1 业务背景
[描述需求背景和当前痛点]

### 1.2 目标
[一句话描述要解决的问题]

### 1.3 成功指标
- 指标 1：...
- 指标 2：...

## 2. 目标用户

| 用户角色 | 描述 | 核心诉求 |
|---------|------|---------|
| 角色 1 | ... | ... |

## 3. 用户故事

### 故事 1
> 作为 [用户角色]，
> 我希望 [功能描述]，
> 以便 [业务价值]。

**验收标准：**
- Given [前置条件] When [操作] Then [预期结果]

### 故事 2
> 作为 [用户角色]，
> 我希望 [功能描述]，
> 以便 [业务价值]。

**验收标准：**
- Given [前置条件] When [操作] Then [预期结果]

### 故事 3
> 作为 [用户角色]，
> 我希望 [功能描述]，
> 以便 [业务价值]。

**验收标准：**
- Given [前置条件] When [操作] Then [预期结果]

## 4. 功能范围

### In Scope（本期包含）
- [ ] 功能点 1
- [ ] 功能点 2

### Out of Scope（本期不包含）
- 功能点 X（后续版本）
- 功能点 Y（待定）

## 5. 数据流 / 业务流程

[文字描述或 ASCII 流程图]

```
用户操作 → 系统处理 → 结果输出
```

## 6. 非功能需求

| 类型 | 要求 |
|------|------|
| 性能 | ... |
| 安全 | ... |

## 7. 依赖与风险

| 依赖/风险 | 类型 | 影响 | 应对措施 |
|---------|------|------|---------|
| ... | 依赖/风险 | 高/中/低 | ... |

## 8. 优先级与里程碑

| 阶段 | 功能点 | 优先级 | 预计完成 |
|------|--------|--------|---------|
| MVP | ... | P0 | ... |
| 后续迭代 | ... | P1 | ... |
```
