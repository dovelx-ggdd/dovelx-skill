## 二、Agent A：API 综合测试

**评级**：⚠️ 部分通过

#### 发现的问题
| 等级 | 描述 |
|------|------|
| 🟠 P1 | PUT /profile：邮箱冲突时返回 500 而非 409 Conflict |
| 🟡 P2 | POST /register：错误响应字段名 `err` 与文档约定 `error` 不一致 |
| 🟡 P2 | PUT /profile：传入 501 字符用户名返回 500 而非 400 |

---

## 三、Agent B：安全测试

**评级**：🟡 有问题

#### 发现的安全问题
| 等级 | 类型 | 描述 |
|------|------|------|
| 🟠 P1 | 缺失速率限制 | POST /login 无登录频率限制，存在暴力破解风险 |
| 🟡 P2 | 信息泄露 | 登录失败区分"用户不存在"/"密码错误"，可枚举用户名 |
| 🟡 P2 | 批量分配 | PUT /profile 提交 `"role":"admin"` 被服务器接受（未过滤） |

---

## 五、Bug 汇总

| 等级 | 来源 | 描述 |
|------|------|------|
| 🟠 P1 | Agent A | PUT /profile 邮箱冲突返回 500 |
| 🟠 P1 | Agent B | 登录接口无速率限制 |
| 🟠 P1 | Agent C | PUT /profile 高并发错误率 2.1% |
| 🟡 P2 | Agent B | 批量分配漏洞（role 字段可写入） |
| 🟡 P2 | Agent A | 超长字符串返回 500 |
| 🟡 P2 | Agent C | 登录错误提示消失过快 |

---

## 六、最终结论

⚠️ **有条件通过** — 修复以下 P1 问题后可上线：
1. PUT /profile 增加邮箱冲突的 409 处理
2. 登录接口添加速率限制（建议 5 次/分钟）
3. 优化 PUT /profile 数据库查询性能
