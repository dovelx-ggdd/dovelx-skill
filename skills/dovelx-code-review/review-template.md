# 代码审查报告模板

生成审查报告时使用此模板。路径：`.claude/doc/<功能名>/code-review-<日期>-v1.md`

```markdown
# 代码审查报告：<功能名>

**审查日期**：YYYY-MM-DD
**审查人**：dovelx-code-review
**审查范围**：
- `path/to/file1.ts`
- `path/to/file2.ts`

## 总体结论

🟢 通过 / 🟡 有条件通过（修复 HIGH 后合并）/ 🔴 阻止合并

> [一句话总结最关键的发现]

## 问题清单

### 🔴 CRITICAL（必须修复）

- `文件路径:行号` — **问题描述**
  - 原因：...
  - 修复建议：...

### 🟠 HIGH（建议修复后合并）

- `文件路径:行号` — **问题描述**
  - 原因：...
  - 修复建议：...

### 🟡 MEDIUM（考虑修复）

- `文件路径:行号` — **问题描述**
  - 修复建议：...

### 🟢 LOW（可选优化）

- `文件路径:行号` — **细节建议**

## 亮点（值得保留的好实践）

- ...

## 修复验证清单

- [ ] 所有 🔴 CRITICAL 问题已修复
- [ ] 所有 🟠 HIGH 问题已修复
- [ ] 重新审查确认无遗漏
```
