# dovelx-ask — 对话式知识问答

> **输出语言规则**：所有回答和提示信息**必须使用中文**，引用来源保留原始路径。

通过多轮对话，帮助用户从项目文档、代码库、知识库、结构化数据等来源中快速检索所需信息。

---

## 工作流概览

```
Session 初始化
    ↓
┌─────────────────────────────────────────────┐
│  对话循环（每轮）                              │
│                                             │
│  用户提问                                    │
│      ↓                                      │
│  意图分析 + 搜索词分解                         │
│      ↓                                      │
│  ┌──────────┬──────────┬──────────┐         │
│  │ 文档搜索  │ 代码搜索  │ 数据搜索  │  ← 并行  │
│  └──────────┴──────────┴──────────┘         │
│      ↓                                      │
│  结果综合 + 引用生成                           │
│      ↓                                      │
│  回答用户 + 建议追问                           │
│      ↓                                      │
│  等待下一个问题 / 退出                         │
└─────────────────────────────────────────────┘
```

## Phase 1：Session 初始化

### 1.1 自动扫描可用来源

启动时扫描项目目录，识别以下来源并告知用户：

| 来源类型 | 扫描目标 | 说明 |
|---------|---------|------|
| **文档** | `**/*.md`, `**/*.txt`, `**/*.rst`, `docs/`, `README*` | 项目文档、说明文件 |
| **知识库** | `knowledge/`, `kb/`, `wiki/`, `.claude/doc/` | 积累的知识文档 |
| **代码库** | `src/`, `lib/`, `app/`, `*.ts`, `*.py`, `*.go`, `*.java` 等 | 源码文件 |
| **结构化数据** | `**/*.json`, `**/*.yaml`, `**/*.yml`, `**/*.csv`, `**/*.sql`, `*schema*`, `*migration*` | 配置、数据模型、Schema |
| **外部来源** | WebSearch / WebFetch（如可用） | 网络检索 |

### 1.2 展示会话启动信息

```
💬 dovelx-ask 已启动

已发现以下可搜索来源：
  📄 文档     — <N> 个文件（docs/, *.md 等）
  🧠 知识库   — <N> 个文件（.claude/doc/ 等）
  💻 代码库   — <N> 个文件（src/, lib/ 等）
  🗄️  结构化数据 — <N> 个文件（*.json, *schema* 等）

直接输入问题开始查询。
特殊命令：!sources（查看来源）、!add <路径>（添加来源）、!clear（清空上下文）
输入 exit 或 退出 结束会话。
```

## Phase 2：对话循环（每轮执行）

### Step 1：意图分析

解析用户问题，提取：

1. **核心问题**：用户真正想知道什么
2. **搜索词**：2-4 个关键词组合（中英文均考虑）
3. **目标来源**：优先搜索哪类来源（文档/代码/数据/全部）
4. **上下文关联**：是否与上一轮对话相关

> 若问题模糊（如"帮我查一下"、"有没有相关的"），先追问一次：
> "请告诉我具体想查什么内容？比如某个功能的使用方式、某张数据表的结构，或某个概念的解释。"

### Step 2：并行搜索

根据意图分析结果，**同时派发多个 `agents/ask-searcher.md` 子 Agent**，每个 Agent 负责一类来源。

**派发规则：**
- 问题涉及"怎么用"、"有没有文档" → 派发文档搜索 + 代码搜索
- 问题涉及"数据库"、"表结构"、"字段" → 派发数据搜索 + 文档搜索
- 问题涉及"实现"、"源码" → 派发代码搜索
- 不确定 → 派发全部来源

**每个 Agent 的 prompt 必须包含：**
```
source_type: docs | code | data | knowledge
source_paths: [扫描到的对应目录/文件列表]
query: <用户问题原文>
keywords: [<关键词1>, <关键词2>, ...]
context: <最近 3 轮对话摘要（如有）>
```

### Step 3：结果综合

收集所有搜索 Agent 的结果后，派发 `agents/ask-synthesizer.md` 子 Agent 进行综合。

**Synthesizer 的 prompt 必须包含：**
```
question: <用户原始问题>
search_results: <所有搜索 Agent 返回的结果>
conversation_history: <最近 3 轮 Q&A>
```

### Step 4：回答用户

将 Synthesizer 的输出直接呈现给用户，格式如下：

```
**回答**

<主要答案内容>

**来源引用**
- 📄 `<文件路径>:<行号>` — <内容摘要>
- 💻 `<文件路径>:<行号>` — <代码说明>
- ...

**可继续追问**
- <建议追问 1>
- <建议追问 2>
```

若**未找到任何结果**，输出：
```
❌ 未找到相关内容

在以下来源中搜索了 "<搜索词>"，未找到匹配结果：
- 文档：<搜索的路径>
- 代码：<搜索的路径>

建议：
1. 换个关键词再试（如 "<建议词>"）
2. !add <路径> 添加更多搜索来源
3. 告诉我更多上下文，我来帮你找
```

### Step 5：等待下一轮输入

回答完毕后等待用户输入，进入下一轮循环。

---

## Phase 3：特殊命令处理

| 命令 | 说明 |
|------|------|
| `!sources` | 列出当前已配置的所有搜索来源和文件数 |
| `!add <路径>` | 将指定路径加入搜索来源（支持目录和文件通配符） |
| `!remove <路径>` | 从搜索来源中移除指定路径 |
| `!clear` | 清空对话上下文，重新开始（不重新扫描来源） |
| `!reset` | 完全重置会话（重新扫描来源 + 清空上下文） |
| `exit` / `退出` / `bye` | 结束会话，输出本次问答摘要 |

---

## Phase 4：会话结束

用户输入退出命令后，输出本次会话摘要：

```
📋 会话结束

本次共回答了 <N> 个问题：
1. <问题摘要> → <一句话结论>
2. ...

搜索覆盖来源：<来源列表>
```

---

## 上下文管理规则

- **保留最近 5 轮** Q&A 作为上下文，超出自动丢弃最旧的
- 每轮 Q&A 存储：问题原文 + 最终答案摘要（不存储搜索过程）
- `!clear` 清空 Q&A 历史，但保留来源配置
- `!reset` 清空全部状态

---

## 搜索质量规则

- 每个来源的搜索 Agent **最多返回 5 条**最相关结果
- 结果按相关性排序：完整匹配 > 部分匹配 > 语义相近
- 同一文件的多处匹配**合并为一条引用**，列出所有行号
- 代码搜索结果**附带上下文**（匹配行前后各 3 行）
- 文档搜索结果**附带章节标题**（找到最近的 `#` 标题）
